## Requirements
### Requirement: 管理员用户 CRUD

系统 SHALL 提供 `/api/admin/users` 路由组，允许具备管理员权限的身份创建、查询、更新和删除用户。创建用户时 MUST 接收用户名、初始密码、显示名和角色，密码在持久化前 MUST 经 bcrypt 哈希并满足密码最小强度。更新操作 SHALL 支持修改显示名、角色、激活状态、用户名，并支持重置密码；修改用户名 MUST 复用用户名唯一性与归一化校验。该路由组 MUST 拒绝 `member` 用户访问，响应 MUST NOT 包含密码哈希。

#### Scenario: 管理员创建用户

- **WHEN** 管理员提交用户名、初始密码、显示名和角色创建用户
- **THEN** 系统创建该用户，密码以 bcrypt 哈希存储，返回不含密码哈希的用户信息

#### Scenario: 管理员更新用户资料

- **WHEN** 管理员修改某用户的显示名或角色
- **THEN** 系统持久化变更并更新 `updated_at`

#### Scenario: 管理员修改用户名

- **WHEN** 管理员将某用户用户名改为一个未被占用的用户名
- **THEN** 系统更新用户名；若目标用户名（归一化后）已被占用则拒绝并返回冲突错误

#### Scenario: 管理员重置用户密码

- **WHEN** 管理员为某用户设置满足强度要求的新密码
- **THEN** 系统用新密码的 bcrypt 哈希覆盖原值，该用户旧密码立即失效

#### Scenario: 管理员停用与启用用户

- **WHEN** 管理员将某用户的激活状态切换为停用
- **THEN** 该用户无法再登录，已签发的 JWT 立即失效，其名下密钥在代理侧失效

#### Scenario: 管理员删除用户

- **WHEN** 管理员删除某个用户
- **THEN** 该用户记录被移除，其名下 API Key 的归属在同一事务内被置空但密钥保留

#### Scenario: member 无法访问用户管理 API

- **WHEN** 角色为 `member` 的用户访问 `/api/admin/users` 任一端点
- **THEN** 系统返回 403

### Requirement: 最后一个管理员的锁定保护

系统 MUST 拒绝停用或删除"系统中最后一个仍处于启用状态的 `admin` 用户"，防止管理员把自己锁在系统之外。`ADMIN_TOKEN` 作为额外的紧急恢复兜底，独立于该保护。

#### Scenario: 拒绝停用最后一个启用的管理员

- **WHEN** 系统中仅剩一个启用的 `admin` 用户，管理员尝试停用或删除该用户
- **THEN** 系统拒绝该操作并提示存在最后管理员约束

#### Scenario: 存在多个管理员时可正常停用

- **WHEN** 系统存在多个启用的 `admin` 用户，管理员停用其中一个
- **THEN** 操作成功

#### Scenario: 全部用户停用后仍可经 ADMIN_TOKEN 恢复

- **WHEN** 数据库中所有用户均被停用
- **THEN** 管理员仍可凭 `ADMIN_TOKEN` 登录并重新启用用户

### Requirement: API Key 所有权分配

系统 SHALL 允许管理员将 API Key 的所有权分配给指定用户或回收为无归属。分配时 MUST 校验目标用户存在。系统 SHALL 支持按用户维度查询其名下的 API Key 数量与列表。

#### Scenario: 分配密钥给用户

- **WHEN** 管理员将某个无归属或已归属的 API Key 分配给一个存在的用户
- **THEN** 该密钥的 `user_id` 更新为目标用户，按用户查询可见该密钥

#### Scenario: 分配给不存在的用户被拒

- **WHEN** 管理员尝试将密钥分配给一个不存在的用户标识
- **THEN** 系统拒绝该操作并返回校验错误

#### Scenario: 回收密钥所有权

- **WHEN** 管理员将某个已归属密钥的归属回收
- **THEN** 该密钥的 `user_id` 被置空，恢复为无归属状态

### Requirement: 用户可用上游配置

系统 SHALL 允许管理员为每个用户配置其可用上游集合，写入 `user_upstreams` 关联表。该集合决定该用户自助创建密钥时能够授权的上游范围。

#### Scenario: 管理员设置用户可用上游

- **WHEN** 管理员为某用户勾选一组可用上游并保存
- **THEN** 系统持久化该用户的可用上游集合，后续该用户自助密钥的授权范围以此为上界

#### Scenario: 回收用户可用上游

- **WHEN** 管理员移除某用户的某个可用上游
- **THEN** 该上游不再可被该用户的新建自助密钥授权

### Requirement: 用户管理后台页面

系统 SHALL 在管理后台提供用户管理页面，以分页方式展示用户列表（含用户名、显示名、角色、激活状态、名下密钥数量），并提供新建用户、编辑资料、修改用户名、重置密码、停用启用、配置可用上游、分配密钥和删除用户的操作入口。最后一个启用 admin 的停用与删除入口 MUST 在界面上被禁用或拦截。页面文案 MUST 在简体中文与英文两套国际化文件中提供。

#### Scenario: 分页展示用户列表

- **WHEN** 管理员打开用户管理页面
- **THEN** 页面按分页展示用户及其用户名、显示名、角色、激活状态和名下密钥数量

#### Scenario: 通过表单创建用户

- **WHEN** 管理员在页面填写新建用户表单并提交
- **THEN** 系统创建用户，列表刷新显示该新用户

#### Scenario: 最后管理员的危险操作入口受限

- **WHEN** 列表中某 `admin` 用户是系统最后一个启用的管理员
- **THEN** 其停用与删除入口被禁用或在提交时被拦截并提示

#### Scenario: 国际化文案完整

- **WHEN** 用户管理页面在简体中文或英文 locale 下渲染
- **THEN** 页面所有文案均有对应翻译，无缺失的占位键
