# upstream-model-allow-rules Specification

## Purpose
定义上游模型允许规则的统一表达与运行时语义，确保显式模型规则、来源标记与导入流程在控制面和代理链路中保持一致。

## Requirements
### Requirement: 上游模型允许规则必须同时支持精确模型、正则模式与人工别名
系统 SHALL 为每个上游提供统一的模型允许规则集合，并同时支持精确模型名、正则模式与人工别名三种规则类型。

#### Scenario: 导入模型目录生成精确规则
- **WHEN** 管理员从上游模型目录中选择若干模型并执行导入
- **THEN** 系统 MUST 将这些模型持久化为当前上游的精确允许规则

#### Scenario: 手工补充正则规则
- **WHEN** 管理员为某上游新增正则模式规则
- **THEN** 系统 MUST 校验规则结构并在保存后用于后续模型匹配

#### Scenario: 手工补充自定义别名
- **WHEN** 管理员为某上游新增人工别名规则
- **THEN** 系统 MUST 允许该别名在运行时映射到该上游的目标模型名

### Requirement: 允许规则必须保留来源与来源差异
系统 SHALL 为导入或录入的模型规则保留来源信息，以区分手工录入、上游原生发现与 LiteLLM 推断候选。

#### Scenario: 从原生发现结果导入规则
- **WHEN** 管理员从上游原生发现目录导入模型
- **THEN** 系统 MUST 为导入规则标记原生发现来源

#### Scenario: 从 LiteLLM 推断目录导入规则
- **WHEN** 管理员从 LiteLLM 推断目录导入模型
- **THEN** 系统 MUST 为导入规则保留推断来源标记
- **AND** 管理端 MUST 能够明确区分该条目与原生发现条目

### Requirement: 运行时必须以显式允许规则决定模型是否可用于某上游
系统 SHALL 在运行时使用显式允许规则判断某个上游是否支持请求模型，并将 exact、regex 与 alias 命中视为允许，未命中则视为不支持。

#### Scenario: 请求模型命中精确规则
- **WHEN** 请求模型名与某上游的精确允许规则完全一致
- **THEN** 该上游 MUST 被视为支持该模型

#### Scenario: 请求模型命中正则规则
- **WHEN** 请求模型名未命中精确规则但命中了某条正则规则
- **THEN** 该上游 MUST 被视为支持该模型

#### Scenario: 请求模型命中人工别名
- **WHEN** 请求模型名匹配某条人工别名规则
- **THEN** 系统 MUST 将其解析到该别名对应的目标模型名
- **AND** 该上游 MUST 被视为支持该模型

#### Scenario: 显式规则未命中
- **WHEN** 某上游显式配置了模型允许规则且请求模型未命中任何精确规则、正则规则或人工别名
- **THEN** 系统 MUST 将该上游视为不支持该模型
