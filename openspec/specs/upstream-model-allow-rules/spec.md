# upstream-model-allow-rules Specification

## Purpose
定义上游模型规则的表达方式、历史字段兼容语义和按上游维度执行的严格匹配要求。

## Requirements
### Requirement: 系统必须支持精确、正则和别名三种模型规则
系统 MUST 为上游提供统一的模型规则集合，至少支持精确模型名、正则模式和人工别名三种规则类型，并在读取和保存时保持规则来源可追踪。

#### Scenario: 精确规则命中
- **WHEN** 请求模型名与某条精确规则完全一致
- **THEN** 系统 MUST 视该上游支持该模型

#### Scenario: 正则规则命中
- **WHEN** 请求模型名满足某条正则规则
- **THEN** 系统 MUST 视该上游支持该模型且保留原始模型名

#### Scenario: 别名规则命中
- **WHEN** 请求模型名命中某条别名规则
- **THEN** 系统 MUST 将目标模型名作为该上游的最终转发模型，并记录本次发生了模型改写

### Requirement: 历史模型配置必须映射为统一规则语义
系统 MUST 将现有 `allowed_models` 与 `model_redirects` 规范化为统一规则视图，使历史数据在迁移窗口内继续保持可读和可用。

#### Scenario: 现有白名单映射为精确规则
- **WHEN** 当前上游配置存在 `allowed_models`
- **THEN** 系统 MUST 将这些模型名解释为精确规则

#### Scenario: 现有重写映射为别名规则
- **WHEN** 当前上游配置存在 `model_redirects`
- **THEN** 系统 MUST 将这些映射解释为别名规则

#### Scenario: 历史字段为空
- **WHEN** 上游既没有规范化规则也没有历史字段
- **THEN** 系统 MUST 将该上游视为未配置显式模型规则

### Requirement: 显式模型规则必须按上游维度严格生效
系统 MUST 在上游维度执行严格模型规则匹配。某个上游存在显式模型规则但请求模型未命中时，该上游 MUST 从候选集中排除；未配置显式规则的上游仍可保持开放候选语义。

#### Scenario: 显式规则未命中时排除该上游
- **WHEN** 某个已授权上游存在显式模型规则且本次请求模型未命中任何规则
- **THEN** 系统 MUST 将该上游从本次候选集中排除

#### Scenario: 未配置显式规则的上游保持开放
- **WHEN** 某个已授权上游未配置任何显式模型规则
- **THEN** 系统 MUST 允许该上游继续参与本次候选选择

#### Scenario: 别名规则改变最终转发模型
- **WHEN** 请求模型命中别名规则
- **THEN** 系统 MUST 将别名目标模型作为该上游的最终转发模型，并将原始模型与解析结果同时记录到路由诊断信息中
