## ADDED Requirements

### Requirement: 系统必须提供后台同步任务注册与调度

系统 MUST 提供后台同步任务注册能力，使每个任务具备稳定任务名、启用状态、执行间隔、启动后首次执行延迟、下次计划执行时间和执行函数。

#### Scenario: 启动后台同步任务

- **WHEN** 应用在 Node.js server runtime 启动且后台同步总开关为开启
- **THEN** 系统 MUST 注册已启用的后台同步任务
- **AND** 系统 MUST 根据每个任务的启动延迟和执行间隔计划首次执行

#### Scenario: 后台同步关闭

- **WHEN** 后台同步总开关为关闭
- **THEN** 系统 MUST 不启动任何后台同步定时器
- **AND** 管理端任务状态 MUST 显示后台同步处于关闭状态

#### Scenario: 跳过非 Node runtime

- **WHEN** 应用运行环境不是 Node.js server runtime
- **THEN** 系统 MUST 不加载后台同步调度器

### Requirement: 系统必须避免同一任务并发执行

系统 MUST 保证同一进程内同一后台同步任务不会并发执行。

#### Scenario: 计划执行遇到任务正在运行

- **WHEN** 某个后台同步任务正在运行且到达下一次计划执行时间
- **THEN** 系统 MUST 跳过本次计划执行
- **AND** 系统 MUST 记录跳过状态和跳过原因

#### Scenario: 手动执行遇到任务正在运行

- **WHEN** 管理员请求手动立即执行某个后台同步任务且该任务正在运行
- **THEN** 系统 MUST 不启动第二个同名任务
- **AND** API MUST 返回当前运行状态

### Requirement: 系统必须持久化后台同步任务状态与执行历史

系统 MUST 持久化每个后台同步任务的最近运行状态与执行历史，使服务重启后管理端仍可查看最近执行结果。

#### Scenario: 任务执行成功

- **WHEN** 后台同步任务执行成功
- **THEN** 系统 MUST 记录最近开始时间、最近完成时间、最近成功时间、执行耗时、成功数量、失败数量和下次计划执行时间
- **AND** 系统 MUST 写入一条成功执行历史

#### Scenario: 任务执行失败

- **WHEN** 后台同步任务执行失败
- **THEN** 系统 MUST 记录最近开始时间、最近完成时间、最近失败时间、失败原因、执行耗时和下次计划执行时间
- **AND** 系统 MUST 写入一条失败执行历史

#### Scenario: 任务部分成功

- **WHEN** 后台同步任务处理多个对象且部分对象失败
- **THEN** 系统 MUST 将任务状态记录为 `partial`
- **AND** 系统 MUST 记录成功数量、失败数量和失败摘要

### Requirement: 管理员必须能查看和手动执行后台同步任务

系统 MUST 提供受管理员认证保护的后台同步任务 API，用于查询任务状态和手动立即执行指定任务。

#### Scenario: 查询任务状态

- **WHEN** 管理员请求后台同步任务列表
- **THEN** API MUST 返回每个任务的任务名、展示名称、启用状态、运行状态、最近开始时间、最近完成时间、最近成功时间、最近失败时间、失败原因、执行耗时和下次计划执行时间

#### Scenario: 手动立即执行任务

- **WHEN** 管理员请求手动立即执行某个已注册后台同步任务
- **THEN** 系统 MUST 立即执行该任务
- **AND** API MUST 返回本次执行结果或当前运行状态

#### Scenario: 拒绝未认证请求

- **WHEN** 请求缺少有效管理员认证
- **THEN** 后台同步任务 API MUST 返回未授权错误

### Requirement: 管理端必须展示后台同步任务状态

管理端 MUST 提供后台同步任务状态面板，使管理员可以查看任务运行状态并手动立即执行任务。

#### Scenario: 展示任务列表

- **WHEN** 管理员打开后台同步任务面板
- **THEN** 页面 MUST 展示每个任务的名称、启用状态、最近状态、下次计划执行时间和手动执行操作

#### Scenario: 展示失败详情

- **WHEN** 某个后台同步任务最近一次执行失败或部分成功
- **THEN** 页面 MUST 展示失败原因或失败摘要

#### Scenario: 手动执行后刷新状态

- **WHEN** 管理员在页面中手动执行某个后台同步任务且执行完成
- **THEN** 页面 MUST 刷新任务状态与相关业务数据

### Requirement: 后台同步任务第一版必须明确单实例边界

系统 MUST 将内置后台同步任务定义为单实例调度能力。

#### Scenario: 单实例部署

- **WHEN** 应用以单个服务实例运行
- **THEN** 后台同步任务 MUST 按配置的时间间隔执行

#### Scenario: 多实例部署

- **WHEN** 应用以多个服务实例运行且每个实例均开启内置后台同步
- **THEN** 系统 MUST 在配置和管理端文案中说明第一版不提供跨实例互斥
