## Why

当前价格目录同步和上游模型目录刷新都依赖管理员手动触发。价格目录滞后会影响后续请求计费准确性，模型目录滞后会让管理端无法及时看到上游新增、下线或变更的模型。

项目已经具备 LiteLLM 价格同步、分层计费规则同步和单个上游模型目录刷新能力；本次变更补齐后台定时执行、任务状态展示和手动立即执行的统一能力。

## What Changes

- 新增后台同步任务能力，支持任务注册、数据库持久化启用开关、动态执行间隔、定时执行、启动后延迟首次执行、任务互斥、手动立即执行、执行状态查询和执行历史记录。
- 新增 `billing_price_catalog_sync` 后台任务，复用现有 LiteLLM 价格同步逻辑，自动刷新 `billing_model_prices` 与 `billing_tier_rules`。
- 新增 `upstream_model_catalog_sync` 后台任务，批量刷新显式开启自动刷新的上游模型目录缓存。
- 扩展上游模型发现配置，使每个上游可以独立控制是否允许后台自动刷新模型目录；手动刷新仍保持可用。
- 新增管理端后台任务状态 API 和界面入口，展示每类后台任务的启用状态、执行间隔、运行状态、最近成功、最近失败、失败原因、执行耗时和下次计划执行时间，并支持管理员动态修改任务配置和手动立即执行。

## Capabilities

### New Capabilities

- `background-sync-tasks`: 定义后台同步任务的配置、调度、状态、历史记录、动态配置和手动立即执行。

### Modified Capabilities

- `billing-price-catalog`: 价格目录必须支持通过后台任务自动同步，并保持失败时保留最近有效价格的语义。
- `upstream-model-discovery`: 上游模型发现配置必须支持显式开启后台自动刷新，并让后台任务只刷新已开启的上游。

## Impact

- 影响后端服务：新增后台任务调度服务、任务注册入口、任务状态查询、配置更新与手动执行 API。
- 影响数据库：需要新增通用后台任务状态与执行历史持久化结构；若模型发现配置需要新增开关字段，同步更新 PostgreSQL 与 SQLite schema、迁移产物和一致性检查。
- 影响管理端：新增后台任务状态面板；上游模型发现配置增加自动刷新开关；Billing 页面继续展示价格同步摘要，并与新任务状态保持一致。
- 影响测试：新增调度服务、任务执行互斥、任务配置更新 API、价格目录自动同步、模型目录自动刷新筛选范围和数据库迁移相关测试。
