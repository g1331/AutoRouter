## ADDED Requirements

### Requirement: 门户上游可见性开关

系统 SHALL 提供一个全局“成员上游可见性”开关，持久化于单例设置行，默认值 MUST 为隐藏。系统 SHALL 提供仅管理员可用的读写端点（`GET/PATCH /api/admin/portal-settings`），非管理员访问 MUST 被拒绝。开关仅影响成员侧（`/api/user/*` 与门户 UI）的上游信息暴露，管理端行为 MUST 不受影响。

#### Scenario: 默认隐藏

- **WHEN** 系统初次读取门户设置且不存在设置行
- **THEN** 系统创建默认设置行，上游可见性为隐藏

#### Scenario: 管理员切换开关

- **WHEN** 管理员通过 PATCH 修改 `expose_upstreams`
- **THEN** 系统持久化新值，后续成员侧请求立即按新值分支

#### Scenario: 非管理员访问被拒

- **WHEN** member 用户或未认证请求访问 portal-settings 端点
- **THEN** 系统返回 403 或 401

### Requirement: 隐藏模式下密钥与授权集自动对齐

隐藏模式下，成员自助密钥的可路由上游集合 MUST 与该用户当前的授权集（user_upstreams）保持一致，由服务端在写路径上物化维护：成员创建密钥时 MUST 绑定当前授权全集；管理员替换某用户授权集时 MUST 同步该用户名下全部密钥的上游集合为新授权集；开关由可见切换为隐藏时 MUST 一次性将所有成员名下密钥重对齐到各自授权全集。代理路由路径 MUST NOT 因该开关引入额外分支。

#### Scenario: 管理员调整授权后密钥跟随

- **WHEN** 隐藏模式下管理员把某用户的授权集从 {A,B} 改为 {A,C}
- **THEN** 该用户名下所有自助密钥的上游集合变为 {A,C}，后续请求只会路由到 A 或 C

#### Scenario: 切换到隐藏时全量对齐

- **WHEN** 管理员把开关从可见切换为隐藏
- **THEN** 所有成员名下密钥的上游集合被重设为各自用户的授权全集

#### Scenario: 空授权集拒绝建键

- **WHEN** 隐藏模式下授权集为空的成员尝试创建自助密钥
- **THEN** 系统拒绝创建并返回明确错误
